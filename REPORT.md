# REPORT

## Overview

This project implements distributed graph algorithms using MPI, including leader election and shortest path computation. The system operates on graphs generated or imported through NetGameSim, partitions the graph across multiple MPI ranks, and performs distributed computation using message passing and collective communication.

The two core algorithms implemented are:
- Leader election using a FloodMax-style approach
- Distributed Dijkstra for shortest paths from a source node

The goal of the project is to demonstrate correct distributed execution, inter-process communication, and the impact of graph structure and partitioning on performance.

---

## Design

### Graph Representation

Graphs are loaded from JSON files generated by NetGameSim. Each graph contains:
- Nodes identified by unique integer IDs
- Directed edges with positive weights

The adjacency list representation is used for efficient traversal and edge relaxation.

---

### Partition Model

The graph is partitioned across MPI ranks using a partition file. Each rank has:

- **Owned nodes**: nodes for which the rank is responsible
- **Ghost nodes**: nodes owned by other ranks but connected to local nodes

This allows each rank to:
- Perform local computation on owned nodes
- Exchange information for cross-rank edges

---

### Communication Model

The system uses MPI collectives for synchronization and communication:

- `MPI_Allreduce` for global agreement (leader election, convergence, min selection)
- `MPI_Allgather` and `MPI_Allgatherv` for exchanging messages between ranks

This ensures all ranks stay synchronized during each iteration of the algorithms.

---

## Algorithm Choices

### Leader Election (FloodMax)

A FloodMax-style algorithm is used for leader election:

1. Each node starts with its own ID as its candidate leader
2. In each iteration:
   - Nodes send their current candidate to neighbors
   - Each node updates its candidate to the maximum received
3. The process repeats until no node changes its candidate

Termination is detected using an MPI collective:
- Each rank tracks whether any local node changed
- `MPI_Allreduce` determines if any rank changed globally

The final leader is the maximum node ID, and agreement is ensured across all ranks.

---

### Distributed Dijkstra

A distributed variant of Dijkstra’s algorithm is implemented:

1. Each rank maintains:
   - Distance values for owned nodes
   - A local priority queue for unsettled nodes
2. Each iteration:
   - Each rank proposes its best local node (minimum distance)
   - All ranks perform a global minimum selection
   - The selected node is settled
3. The owning rank relaxes outgoing edges:
   - Local neighbors are updated directly
   - Remote neighbors are updated via message passing

This continues until no more reachable nodes remain.

The algorithm uses collective communication each iteration to ensure correctness and synchronization.

---

## Experimental Setup

Experiments were conducted using:

- A medium-sized graph (~10–100 nodes)
- Partitioned across 2 MPI ranks
- A perturbed version of the same graph to simulate changes in structure

All runs were executed using:

```bash
mpirun -np 2 ./ngs_mpi --graph <graph> --part <partition> --algo <leader|dijkstra> [--source 0]
```

### Analysis

Overall, the experiment showed that increasing the number of nodes did not break either algorithm. Both leader election and Dijkstra remained correct on both smaller and larger graphs. The main difference was in cost: larger graphs increased communication volume, total work, and runtime, while smaller graphs completed more quickly with fewer iterations and messages. This demonstrates that the implementation is stable across graph sizes and that the observed scaling effects are primarily performance-related rather than correctness-related.
